---

## 1. 业务类型

AI 驱动的 B2B 漫剧代加工生成平台

## 2. 目标市场

- 一期：海外美区 TikTok
- 二期：国内抖音（规则做成可插拔插槽，兼容二期其他平台）

## 3. 目标客户

B2B 代工——客户提供剧本，平台交付稳定、风格一致、返工少的成片

## 4. 目标交付物

- 单集成片：50-60s 竖屏 9:16 漫剧 MP4
- 批量生产系统：支持日产 10-20 集（A级标准量产）
- Web UI：归档分析、二次剪辑、审核、平台打回修改
- 多版本导出：TikTok 引流版 / Series 付费版 / App 分发版

## 5. 用户能力边界

- 单兵建设（如意粑粑），懂代码和 AI，不懂漫剧专业制作流程
- 剪辑师参与返工环节
- 系统需要把影视工业知识结构化、参数化、自动化

## 6. 成功指标

| 指标 | A级 | S级 | SSS级 |
|------|-----|-----|-------|
| 角色跨集一致性 | > 0.80 | > 0.90 | > 0.95 |
| 音画同步偏差 | < 0.1s | < 0.05s | < 0.02s |
| 单集成本 | ≤ $1 | ≤ $5 | ≤ $15 |
| 日产能力 | 10-20集 | 5-10集 | 1-3集 |
| 非必要人工介入 | < 15% | < 10% | < 5% |

## 7. 已有约束

- 预算：性价比优先（GPU 走 Vast.ai，API 质量优先但开源达标则首选）
- 时间：1 周 Demo 落地
- 团队：单兵建设 + 剪辑师返工
- 算力：Vast.ai 按需 GPU 租赁
- 平台规则（TikTok 一期）：竖屏 9:16、AIGC 标识、C2PA、版权音乐限制、禁用亚裔角色/黑人反派/东方元素
- 技术架构偏好：微服务 + 消息队列、三层结构（便宜稳定层/资产生产层/高光精修层）、OpenClaw 多 Agent 编排 + 自建调度层
- Demo 可用剧本：有现成 22 集完整剧本
