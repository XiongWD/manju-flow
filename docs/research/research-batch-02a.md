## 1. Kling 3.0 / 2.0 Master

### 官方定价表 ✅ 已验证事实

| 指标 | 官方价格 | 说明 |
|------|---------|------|
| 每秒费用（音频关闭） | $0.07/s | 5s ≈ $0.35；10s ≈ $0.70 |
| 每秒费用（音频开启） | $0.14/s | 5s 含音频 ≈ $0.70；10s ≈ $1.40 |
| 视频时长选项 | 5s / 10s / 15s | 3.0 版本支持最长 15s |
| 分辨率 | 最高 1080p | 支持 16:9, 9:16, 1:1 |
| 第三方代理价（klingapi.com） | 官方价 70% | 音频关 $0.049/s，音频开 $0.098/s |

> 来源：klingapi.com/pricing（第三方代理），klingai.com 官方定价页。✅ 已验证事实

### 核心能力评分

| 能力 | 评分(1-10) | 说明 |
|------|-----------|------|
| 角色一致性/锁定 | **9/10** | 支持多图/视频主体引用（Subject Reference），3.0 Omni 版本进一步强化 |
| 口型同步（Lip Sync） | **9/10** | 原生音画同步，支持多语言（中英日韩西）+ 方言口音混合 |
| 最大时长 | **8/10** | 3.0 版本支持 15s 单次生成，视频延展可达 3 分钟 |
| 智能分镜 | **9/10** | AI 导演系统自动调度镜头类型、机位、正反打 |
| 原生音频 | **9/10** | 角色驱动音色、多语言混合、环境音效同步 |
| 文字生成 | **8/10** | 原生中英文文字渲染，支持标牌/字幕 |
| API 可用性 | **9/10** | 完整 API（T2V、I2V、运动控制），文档齐全 |

### 漫剧适用度判断：⭐⭐⭐⭐⭐ 极高

**理由：**
- 15s 超长生成对漫剧单镜头足够
- 原生音画同步 + 多语言口型 = 漫剧对话核心需求
- 角色一致性能力业界领先（Subject Reference）
- 智能分镜系统可自动处理正反打，大幅降低人工编排
- 每秒 $0.07-0.14 的价格在商业模型中极具竞争力

⚠️ 需注意：3.0 版本标注为"高级内测中"，正式版 API 可能有变动

| 能力 | 评分(1-10) | 说明 |

| API 可用性 | **9/10** | 完整 API，支持 T2V + I2V，2-10s 时长 |

### 漫剧适用度判断：⭐⭐⭐⭐ 高（有缺口）

**理由：**
- 角色一致性极强，适合漫剧多场景角色锁定
- 物理模拟顶级，动作场景表现优秀
- $0.05/s 的 Turbo 价格是商业模型中最便宜之一
- **缺口：无原生口型同步/音频**，漫剧对话需要后期配音或使用 Characters API
- Gen-4.5 更强但价格翻倍（$0.12/s）

🆕 增量机会：Characters API 可能提供对话式口型同步，值得后续深入测试

- 单价较高（$0.15-0.40/s），大量生成成本显著
- Ultra 订阅 $249.99/月才能获得合理生成量

🆕 增量机会：如需高品质音频场景，Veo 3.1 可作为音频层补充工具

## 5. Wan 2.1（开源）

### 部署成本与能力 ✅ 已验证事实

**模型规格：**

| 模型 | 参数量 | VRAM 需求 | 分辨率 | 推理速度（RTX 4090） |
|------|--------|----------|--------|---------------------|
| T2V-1.3B | 1.3B | 8.19 GB | 480p | ~4 分钟/5s |
| T2V-14B | 14B | 24+ GB | 480p/720p | 更长 |
| I2V-14B-720P | 14B | 24+ GB | 720p | - |
| I2V-14B-480P | 14B | 24+ GB | 480p | - |
| FLF2V-14B | 14B | 24+ GB | 720p | - |
| VACE-1.3B/14B | 1.3B/14B | 8-24+ GB | 480p-720p | - |

**自部署 GPU 成本估算（云租赁）：**

| GPU | 显存 | 适合模型 | 云端时租（估算） |
|-----|------|---------|----------------|
| RTX 4090 | 24 GB | 1.3B（推荐）/ 14B（需 offload） | $0.30-0.50/h |
| A100 40GB | 40 GB | 14B（舒适运行） | $1.50-2.00/h |
| A100 80GB | 80 GB | 14B + 多任务 | $2.50-3.50/h |
| H100 | 80 GB | 14B 最优 | $3.00-4.00/h |

**每秒成本估算（14B 模型 @ A100）：**
- 5 秒 720p 视频 ≈ 3-5 分钟生成
- GPU 成本 ≈ $0.08-0.17/视频秒（含生成时间）
- 1.3B 模型 @ RTX 4090 更便宜：≈ $0.02-0.04/视频秒

**核心能力：**

| 能力 | 评分(1-10) | 说明 |
|------|-----------|------|
| 画质 | **7/10** | 14B 模型 720p 质量不错，但不如闭源模型 |
| 文字渲染 | **8/10** | 首个支持中英文文字生成的视频模型 |
| 开源生态 | **10/10** | Apache 2.0 许可（部分），完全可商用 |
| ComfyUI 集成 | **10/10** | 2025-02-27 已集成，完整支持 T2V/I2V/FLF2V/VACE |
| 扩展性 | **9/10** | VACE 编辑、DiffSynth-Studio、LoRA 训练、FP8 量化 |
| 口型同步 | **3/10** | 无原生口型同步，需外部工具 |
| 角色一致性 | **5/10** | 无内置角色锁定，需 LoRA 或外部方案 |

**社区衍生项目：**
- Helios：分钟级长视频（单 H100 约 19.5 FPS）
- LightX2V：RTX 5090/4060(8GB) 可运行
- Phantom：多主体引用生成
- EchoShot：多镜头人物一致性

> 来源：github.com/Wan-Video/Wan2.1, reddit.com/r/StableDiffusion, digitalocean.com ✅ 已验证事实

### 漫剧适用度判断：⭐⭐⭐ 中等（需工程投入）

**理由：**
- **成本优势明显**：1.3B 模型每秒成本仅为商业模型的 1/3-1/5
- ComfyUI 集成成熟，可构建自动化流水线
- **严重缺口：无口型同步、无角色锁定**
- 720p 分辨率可能不够（漫剧需要 1080p）
- 需要工程团队搭建和维护自部署环境
- 适合做**视觉底板生成**，音频/口型需其他工具补充

🆕 增量机会：Phantom + EchoShot 组合可能解决角色一致性，值得后续评估

### 漫剧项目推荐优先级

1. **Kling 3.0** — 全能型首选，音画同步 + 角色锁定 + 智能分镜，最适合对话驱动的漫剧
2. **Runway Gen-4 Turbo** — 性价比最高，角色一致性顶级，但缺音频需后期补

5. **Pika 2.5** — 最便宜入门选项，但口型同步缺失对漫剧是硬伤
