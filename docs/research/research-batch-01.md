## 一、ArcReel

### 1.1 定位与核心功能

**定位**: 开源 AI 视频工作台，专注「小说→成片」全链路 Agent 编排。

**面向谁**: 独立创作者、短剧/漫剧工作室、内容批量化生产团队。

**核心功能列表**:
- 小说原文输入 → Agent 自动拆解剧本、分集规划
- 三类资产库建立（角色、场景、道具），跨镜头一致性追踪
- 角色设计图先行，作为后续分镜/视频生成基准（"角色 DNA"）
- 异步任务引擎（RPM 限速、lease-based 调度、断点续传）
- 版本管理（一键回滚、多版本并排对比）
- 剪映 5.x/6+ 草稿直出 + FFmpeg 全自动合成
- 多模型矩阵：Gemini、火山方舟、Grok、OpenAI 四家预置 + 自定义端点

✅ **已验证事实** — 来源：ArcReel - https://github.com/ArcReel/ArcReel + https://arc-reel.com/

### 1.2 定价模型

- **开源版**: AGPL-3.0 协议，永久免费，Docker Compose 自部署
- **商业版**: v0.9 内测中，提供托管基础设施、批量采购供应商额度、优先队列、SSO、白标定制、SLA
- **用户成本**: 仅为上游 AI 供应商费用（Gemini/火山方舟/Grok/OpenAI）
- 无独立定价页（404），商业版通过内测申请制

✅ **已验证事实** — 来源：ArcReel - https://arc-reel.com/（FAQ 部分）

### 1.3 能力边界

**能做到**:
- 小说→结构化剧本→分镜图→视频片段的完整链路
- 跨镜头角色/场景一致性（设计图先行方案）
- 多供应商自动切换与自定义端点接入
- 剪映草稿导出（留手工精剪空间）

**做不到**:
- 无 ComfyUI 集成（从代码和文档中未发现 ComfyUI 相关模块）
- 无 LoRA 训练/微调能力（角色一致性依赖"设计图+prompt"方案，非 LoRA）
- 无自动音频/配乐生成（需外部处理）
- 仅支持 POSIX 环境，无 Windows 原生支持

⚠️ **有争议结论** — 来源：ArcReel - GitHub 代码分析。ComfyUI 和 LoRA 相关结论基于代码搜索未发现，可能在私有商业版中存在。

### 1.4 缺口分析（对 B2B 漫剧代工平台）

| 缺口 | 说明 |
|------|------|
| 无多租户/B2B 架构 | 面向单用户/小团队，无客户隔离、权限管理、工单系统 |
| 无 ComfyUI 集成 | 无法利用 ComfyUI 生态的 ControlNet/IPAdapter 等精细控制 |
| 无 LoRA 角色一致性 | 角色一致性依赖 prompt+设计图，无法做到像素级锁定 |
| 无批量任务调度 | 虽有异步引擎但无优先级队列、并发控制、成本核算 |
| 无客户交付界面 | 无审片、批注、交付确认等 B2B 协作流程 |
| 无音频/配乐自动生成 | 需要外部 TTS/配乐方案 |

### 1.5 可复用点

1. **Subagent 编排架构**: 主 Agent 分派任务给聚焦 Subagent，上下文不爆 → 可借鉴其 Skill + Subagent 分工模式
2. **三类资产库设计**: 角色/场景/道具的结构化管理 → 直接复用其资产库 schema 设计思路
3. **异步任务引擎**: RPM 限速、lease-based 调度、断点续传 → 可参考其任务队列架构
4. **统一后端协议**: 一套协议覆盖图像/视频/文本三类生成 → 可借鉴多供应商抽象层设计
5. **剪映草稿导出**: 按集导出剪映兼容草稿包 → 可复用其剪映协议实现

🆕 **增量机会** — 来源：用户资料。以上可复用点为基于 ArcReel 架构分析的推断。

## 二、OpenMontage

### 2.1 定位与核心功能

**定位**: 全球首个开源 agentic 视频制作系统，12 条 pipeline、52 个工具、500+ agent skills。

**面向谁**: AI coding assistant 用户（Claude Code/Cursor/Copilot/Windsurf/Codex），视频内容创作者。

**核心功能列表**:
- 12 条生产 pipeline：Animated Explainer、Animation、Avatar Spokesperson、Cinematic、Clip Factory、Documentary Montage、Hybrid、Localization & Dub、Podcast Repurpose、Screen Demo、Talking Head
- 52 个生产工具：视频生成、图像创建、TTS、音乐、音频混合、字幕、增强、分析
- 400+ agent skills：生产技巧、pipeline 导演、创意技术、质量检查清单
- 参考视频驱动创作（Paste A Video → 分析 → 差异化方案）
- 真实素材纪录片创作（CLIP 搜索 Archive.org/NASA/Wikimedia 免费素材）
- 实时 Web 研究内置（15-25+ 搜索 → 结构化研究简报）
- 双渲染引擎：Remotion（React）+ HyperFrames（HTML/GSAP）
- 7 维度供应商评分选择器
- 生产级质量门禁（ffprobe 验证、帧采样、音频分析、交付承诺检查）

✅ **已验证事实** — 来源：OpenMontage - https://github.com/calesthio/OpenMontage

### 2.2 定价模型

- **开源协议**: AGPL-3.0
- **零成本路径**: Piper TTS（免费离线）+ Archive.org/NASA/Wikimedia 免费素材 + Pexels/Unsplash/Pixabay 免费 key
- **低成本路径**: ~$0.15-$1.50/视频（FLUX 图像 + Kling/Veo 视频）
- **全配置路径**: ~$1-$3/视频
- **GPU 本地**: 支持 wan2.1、hunyuan、ltx2、cogvideo 等本地模型

✅ **已验证事实** — 来源：OpenMontage - GitHub README

### 2.3 能力边界

**能做到**:
- 端到端视频制作（从创意到成片）
- 真实素材检索+编辑（非仅图片动画）
- 多 pipeline 覆盖（解说、动画、头像、电影、播客、本地化等）
- Agent-first 架构（AI coding assistant 即编排器）
- 零 API key 也能产出视频
- 参考视频分析→差异化方案生成

**做不到**:
- 无漫剧/短剧专用 pipeline（面向通用视频制作）
- 无跨镜头角色一致性机制（无"角色 DNA"概念）
- 无小说→剧本→分镜的结构化拆解
- 无批量项目管理/多租户
- 无 ComfyUI 集成
- 依赖 AI coding assistant 作为编排器（非独立 Web 应用）

⚠️ **有争议结论** — 来源：OpenMontage - GitHub 分析。以上"做不到"基于当前公开代码，未来可能扩展。

### 2.4 缺口分析

| 缺口 | 说明 |
|------|------|
| 无漫剧专用 pipeline | 12 条 pipeline 均为通用视频，无漫画/漫剧场景 |
| 无角色一致性 | 无跨镜头角色追踪、设计图基准机制 |
| 无小说/剧本拆解 | 无长文本→结构化剧本的专用流程 |
| 非 Web 应用 | 依赖 AI coding assistant，无独立 UI |
| 无 B2B 协作 | 无多租户、权限、审片、交付流程 |
| 无 ComfyUI/LoRA | 无法利用开源图像生态做精细控制 |

### 2.5 可复用点

1. **Agentic Pipeline 架构**: YAML pipeline 定义 + Markdown stage director skills → 可直接借鉴其 pipeline 模块化设计
2. **工具注册表 (tool_registry)**: 52 工具统一发现与能力封装 → 可参考其工具抽象层
3. **7 维度供应商评分**: 任务适配、输出质量、控制力、可靠性、成本效率、延迟、连续性 → 可借鉴其供应商选择算法
4. **多渲染引擎选择**: Remotion vs HyperFrames 按场景自动选择 → 可参考其渲染引擎抽象
5. **生产级质量门禁**: ffprobe 验证 + 帧采样 + 音频分析 + 交付承诺检查 → 可直接复用其质检思路
6. **CLIP 素材检索**: 语义搜索免费素材库 → 可借鉴其素材检索架构
7. **成本预估机制**: 执行前成本估算 + spend caps → 可参考其成本控制设计

🆕 **增量机会** — 来源：用户资料。OpenMontage 的 agentic pipeline + tool registry + 质量门禁是最值得借鉴的架构。

## 三、video-use

### 3.1 定位与核心功能

**定位**: 用 coding agent 对话式编辑视频的开源 skill，专注后期剪辑。

**面向谁**: 使用 Claude Code/Codex/OpenClaw 等 coding agent 的视频创作者。

**核心功能列表**:
- 对话式视频编辑（聊天→剪辑方案→执行）
- 自动转录（ElevenLabs Scribe，word-level 精度）
- 剪辑决策：去 filler words、去死时间、多 take 选择
- 自动调色（per-segment ffmpeg 滤镜链）
- 字幕生成（可定制 chunking/case/placement）
- 动画叠加（Manim/Remotion/PIL，parallel sub-agents）
- **Self-evaluation 循环**: 渲染后在每个剪切点验证视觉连续性、音频爆音、字幕遮挡
- 会话记忆持久化（project.md）
- 30ms 音频淡入淡出（Hard Rule）

✅ **已验证事实** — 来源：video-use - https://github.com/browser-use/video-use + SKILL.md

### 3.2 定价模型

- **开源协议**: 未明确标注（GitHub 显示 no license）
- **依赖**: ElevenLabs API（Scribe 转录，需付费 key）
- **免费组件**: ffmpeg、PIL、Manim（开源）
- **无独立定价**: 纯开源 skill，无 SaaS 层

⚠️ **有争议结论** — 来源：video-use - GitHub。license 信息缺失，可能为 proprietary。

### 3.3 能力边界

**能做到**:
- 精准的视频后期剪辑（word-level 精度）
- 自动化剪辑决策（filler removal、dead space removal）
- 自动调色、字幕、动画叠加
- 渲染后 self-evaluation（视觉/音频/字幕三维质检）
- 多 take 最佳选择（editor sub-agent brief）

**做不到**:
- 无视频生成能力（纯后期工具）
- 无图像生成/角色设计
- 无剧本/分镜拆解
- 无漫剧/短剧专用流程
- 无批量处理/项目管理
- 依赖 ElevenLabs API（非完全免费）

### 3.4 缺口分析

| 缺口 | 说明 |
|------|------|
| 纯后期工具 | 无前端生成能力，需配合其他工具 |
| 无漫剧场景 | 面向真人视频剪辑，非漫画/动画 |
| 无批量能力 | 单视频对话式编辑，无并行项目 |
| 无 B2B 协作 | 无审片、批注、交付流程 |
| API 依赖 | ElevenLabs 非免费 |

### 3.5 可复用点

1. **Self-evaluation 循环**: 渲染后在每个剪切点运行 timeline_view 验证 → **可直接借鉴用于漫剧质检**
2. **Hard Rules 设计模式**: 将生产正确性规则（非创意偏好）硬编码为不可协商规则 → 可借鉴其规则分层思路
3. **Ask → Confirm → Execute → Self-Eval → Persist 流程**: 人机协作五步法 → 可复用于漫剧制作流程
4. **Editor Sub-agent Brief**: 结构化的剪辑决策 prompt 模板 → 可借鉴其多 take 选择算法
5. **Per-segment 处理架构**: 分段提取→处理→拼接（非单 pass）→ 可参考其分段渲染架构
6. **Audio-first 剪辑**: 以音频边界为剪切候选 → 可借鉴其音频驱动思路

🆕 **增量机会** — 来源：用户资料。video-use 的 self-evaluation 循环和 Hard Rules 设计模式是最大可借鉴价值。

## 四、aidrama

### 4.1 定位与核心功能

**定位**: 闭源 AI 影视创作与协作平台（智象未来/HiDream 旗下），专注"AI影视精品化、工业化、规模化制作"。

**面向谁**: 影视制作团队、短剧/漫剧工作室、广告/教育内容团队。

**核心功能列表**:
- 全链 AI "控场"：剧本解析→分镜→镜头生成→快剪→成片
- 导演级控镜：精准掌控运镜、光影、构图
- 分镜+画布：专业分镜表 + 高自由度画布整合创意
- 工业化协同：权限管理、在线批注、人效分析
- 项目全程管理：立项→交付全流程，多项目并行、审片、核算、版本回溯
- 全量资产留存：素材/分镜/成片自动归档，角色场景一键复用
- 多场景覆盖：短剧/漫剧、绘本/漫画、教育培训、广告TVC、游戏

✅ **已验证事实** — 来源：aidrama - https://aidrama.hidreamai.com/

### 4.2 定价模型

- **闭源 SaaS**: 需登录使用，手机号/微信登录
- **定价信息**: 官网未公开定价页，推测为按需付费或企业定制
- **无 API**: 无公开 API 或开发者文档

⚠️ **有争议结论** — 来源：aidrama - 官网分析。定价信息不可见，仅为推测。

### 4.3 能力边界

**能做到**:
- 短剧/漫剧工业化生产（案例有蒸汽朋克、都市追逐等复杂场景）
- 多角色协作与权限管理
- 工业级画面质量（3A 级视效案例）
- 全流程项目管理（立项→交付）
- 资产归档与复用

**做不到**:
- 无开源/自部署方案
- 无公开 API（无法集成到外部系统）
- 无 ComfyUI/自定义模型接入
- 依赖平台锁定（数据/资产在平台内）
- 可能无 LoRA/自定义训练能力

### 4.4 缺口分析

| 差异点 | aidrama | 我们的定位 |
|--------|---------|-----------|
| 架构 | 闭源 SaaS | 开源/可自部署 |
| 集成 | 无 API | API-first |
| 模型 | 平台内置 | 多供应商+自定义 |
| 定位 | 通用影视制作 | 漫剧代工垂直 |
| 协作 | 平台内协作 | B2B 外部交付 |
| 自定义 | 有限 | ComfyUI/LoRA/自定义模型 |

### 4.5 可复用点

1. **分镜+画布的设计思路**: 专业分镜表 + 高自由度画布 → 可借鉴其分镜 UI 设计
2. **资产归档与复用**: 素材/分镜/成片自动归档 → 可参考其资产管理系统
3. **项目全程管理**: 立项→交付全流程覆盖 → 可借鉴其项目管理模块设计
4. **工业化协同**: 权限管理、在线批注、人效分析 → 可参考其 B2B 协作功能
5. **"导演级控镜"理念**: 精准掌控运镜、光影、构图 → 可借鉴其镜头控制参数设计

🆕 **增量机会** — 来源：用户资料。aidrama 作为闭源竞品，其工业化协同和项目管理是 B2B 场景的重要参考。
