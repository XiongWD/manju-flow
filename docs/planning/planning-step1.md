## 🎯 Demo 阶段验证项（必须在 Phase 1 启动前完成）

### #1 — IP-Adapter Plus A 级一致性验证

| 要素 | 内容 |
|------|------|
| **优先级** | P0 |
| **问题** | IP-Adapter 6.5/10 是否满足 A 级一致性 ≥0.80 的要求？ |
| **测试方法** | 选取 3 个风格各异的漫剧角色（男/女/非人类），每个角色生成 5 个不同场景镜头（正面/侧面/远景/特写/动作），共 15 张图。由 2 名评测者盲审打分（0-1 一致性评分），取平均值。 |
| **通过标准** | 平均一致性评分 ≥ 0.80（满分 1.0）；单角色最低分 ≥ 0.70 |
| **所需资源** | ComfyUI 环境 + IP-Adapter Plus 模型 + 3 组角色参考图 + 2 名评测者（可由建设者+1 人承担） |
| **预计耗时** | 2-3 小时（环境配置 30min + 生成 1h + 评测 1h） |
| **阻塞 step2** | ✅ **阻塞**。若不通过，A 级流水线需降级为 B 级演示，Phase 拆解方案需重写。 |
| **失败预案** | 一致性 < 0.80 → A 级流水线降级为 B 级（仅演示概念），或改用 Kling 3.0 角色锁定（6.5/10 但原生集成）。 |

---

### #2 — LoRA → Seedance 2.0 Fast 衔接质量验证

| 要素 | 内容 |
|------|------|
| **优先级** | P0 |
| **问题** | LoRA 微调生成的角色图 → Seedance 2.0 Fast img2vid 的衔接质量是否可控？ |
| **测试方法** | 用 LoRA 训练 1 个测试角色（10-20 张参考图），生成 3 张角色图作为 img2vid 输入，提交 Seedance 2.0 Fast 生成 5-8s 视频 clip。评估：①角色特征保留度 ②画面流畅度 ③风格一致性。共测 3 组（3 张×3 次=9 个 clip）。 |
| **通过标准** | 角色特征保留度 ≥ 7/10（盲审）；无明显变形/闪烁；风格与原图偏差 ≤ 20% |
| **所需资源** | LoRA 训练环境（Vast.ai GPU） + Seedance 2.0 API Key + 训练素材 10-20 张 + 评测者 |
| **预计耗时** | 4-6 小时（LoRA 训练 2h + 图生 30min + img2vid 提交等待 1-2h + 评测 1h） |
| **阻塞 step2** | ✅ **阻塞**。SSS 路径 B（默认路径）依赖此衔接；若失败需触发降级链，Phase 拆解需调整。 |
| **失败预案** | 衔接不可控 → 触发降级链：路径 A（Kling $14-18）→ SSSS（IP-Adapter+Seedance $5-7）→ A 级兜底。每级保留上一版本资产，降级无需重训。 |

### #3 — Kling 内容审核误判率验证

| 要素 | 内容 |
|------|------|
| **优先级** | P1 |
| **问题** | Kling 内容审核对漫剧角色的误判率？ |
| **测试方法** | 准备 5 个典型漫剧角色设计稿（含夸张表情、战斗场景、奇幻元素），分别提交 Kling 3.0 img2vid 接口，记录审核通过/拒绝/需修改的比例。每角色提交 2 次（不同场景），共 10 次提交。 |
| **通过标准** | 审核通过率 ≥ 80%（10 次中至少 8 次通过）；无永久封禁类拒绝 |
| **所需资源** | Kling 3.0 API Key + 5 个角色设计稿 + 10 个场景 prompt |
| **预计耗时** | 1-2 小时（素材准备 30min + 提交等待 30-60min + 结果整理 15min） |
| **阻塞 step2** | ✅ **阻塞**。若误判率 > 20%，Demo 阶段 A 级流水线需切换为 Seedance 2.0，Phase 拆解需调整。 |
| **失败预案** | 误判率高 → 切换 Seedance 2.0 Fast 作为 Demo 阶段视频生成方案（step0 已预留）。 |

### #4 — Suno Pro 商用许可 TikTok 覆盖验证

| 要素 | 内容 |
|------|------|
| **优先级** | P1 |
| **问题** | Suno Pro 商用许可是否覆盖 TikTok 发布？ |
| **测试方法** | 查阅 Suno Pro 最新许可条款（Terms of Service + Commercial License），重点检查：①是否允许社交媒体平台分发 ②是否覆盖 TikTok/短视频平台 ③是否需要额外标注。同时查阅社区案例和官方 FAQ。 |
| **通过标准** | 许可条款明确允许 TikTok 平台发布；或官方确认无限制 |
| **所需资源** | 网络访问（Suno 官网 + 许可条款页面）；无需 GPU 或 API |
| **预计耗时** | 30-60 分钟（条款阅读 + 社区搜索 + 整理结论） |
| **阻塞 step2** | ✅ **阻塞**。UC-4 已确认默认用 Veo 3.1 原生音频，但若 Suno 许可明确覆盖 TikTok，可作为 BGM 备选方案纳入 Phase 拆解。 |
| **失败预案** | 许可不明确或不覆盖 → 严格按 UC-4 执行：默认 Veo 3.1 原生音频，Suno 推迟至 Step 6。 |
