## 2. AiDrama（帧赞）详细分析

> 来源：https://aidrama.hidreamai.com/ 实际页面抓取

### 2.1 整体定位
- **Slogan**: "为专业团队而生的AI影视创作引擎 · 每一帧都很赞"
- **目标用户**: 专业影视制作团队
- **核心卖点**: 工业级成片、导演级控镜、多模态画布、全流程管理

### 2.2 色彩方案
- **主背景**: 深黑 (#0A0A0A ~ #141414)
- **主色**: 紫蓝色调（#7C3AED 系列渐变）
- **强调色**: 影视级紫色光效、渐变紫蓝按钮
- **文字**: 白色标题 + 浅灰正文
- **卡片**: 深灰底 (#1A1A1A) + 微弱边框

### 2.3 布局特征
- 首页全幅视频/图片封面展示（视频缩略图带播放按钮）
- 大尺寸项目展示卡片（4:3 或 16:9 宽画幅）
- 分区块滚动：Hero → 能力介绍 → 产品核心 → 作品展示 → 底部
- **能力介绍区**: 6列网格，每格 icon + 标题 + 2行描述
- **作品展示区**: 大图卡片 + overlay 播放按钮 + 标题描述

### 2.4 排版特征
- 标题用大号中文字体（36-48px），bold
- 副标题/描述用浅灰色（#A3A3A3系列）
- 功能标签用圆角小标签（如"多模态共存 自由画布"）
- 中英文混排：功能名中文，术语保留英文

### 2.5 交互特色
- 视频封面 hover 播放预览
- 渐变按钮 CTA（"申请试用"、"立即登录"）
- 底部产品矩阵横排（智小象AI、智小剪AI、vivago.ai、API）

### 2.6 对漫剧 OS 的启示
- ✅ 深黑背景 + 紫蓝主色 = 影视工业感
- ✅ 项目卡片用大图封面展示
- ✅ 功能区块用 icon + 短标题 + 描述
- ✅ 中英文混排策略（中文标题 + 英文术语）
- ⚠️ 首页营销页风格，内页编辑器风格可能不同（需参考其产品内页）

## 3. Runway 详细分析

> 来源：https://runwayml.com/ + https://app.runwayml.com/ + DESIGN.md

### 3.1 整体定位
- **Slogan**: "Building AI to Simulate the World"
- **目标用户**: 影视创作者、VFX 艺术家
- **核心卖点**: Gen-4.5 视频生成、GWM 世界模型、Characters 实时视频代理

### 3.2 色彩方案（从 DESIGN.md 提取）
- **主背景**: Runway Black (#000000) / Deep Black (#030303)
- **卡片/容器**: Dark Surface (#1A1A1A)
- **边框**: Border Dark (#27272A) — 极度不可见
- **主文字**: Pure White (#FFFFFF)
- **次级文字**: Cool Slate (#767D88) / Mid Slate (#7D848E)
- **功能色**: 几乎没有 — 纯黑白灰，色彩来自内容本身
- **按钮**: 透明底+细微边框 或 纯文字链接

### 3.3 排版系统
- **单字体**: abcNormal（替代：Inter / DM Sans）
- **Display 48px**, weight 400, line-height 1.0, letter-spacing -1.2px
- **Section 40px**, weight 400, line-height 1.0-1.1
- **Body 16px**, weight 400-600, line-height 1.3-1.5
- **Caption 14px**, weight 500-600, UPPERCASE, letter-spacing 0.35px
- **Micro 11px**, weight 450

### 3.4 导航模式
- **首页**: 水平极简导航，透明叠加在影视 Hero 上
- **App 内页**: 左侧简洁 App 分类面板 + 中间工作区
- **App 选择页**: 卡片网格，每张大图+标题+简述

### 3.5 布局与内容展示
- 全幅影视级图片/视频 Hero（占据整个 viewport）
- 杂志式不对称图片网格（大图+小图混排）
- 卡片即图片 — 图片本身就是卡片
- 零阴影，零装饰边框 — UI 退到无形
- Editor 页面：左侧 App 列表 → 中间工作区 → 右侧参数面板

### 3.6 交互特色
- Hover 时文字变白/更高不透明度
- 极细微的过渡（0.2s ease）
- 视频/图片 hover 轻微放大
- 卡片圆角 8px，按钮 4px
- 全幅 Cinematic 图片作为页面分割
- Uppercase 标签做导航结构

### 3.7 对漫剧 OS 的启示
- ✅ **零阴影哲学** — 深度靠背景色层差，不靠 shadow
- ✅ **UI 退到无形** — 让内容（项目封面/分镜/视频）成为视觉主角
- ✅ **Tight 排版** — 标题 line-height 1.0, negative letter-spacing
- ✅ **冷灰次级文字** — 不用暖灰，用 #767D88 这种蓝灰色
- ✅ **单字体系统** — 简化到极致
- ⚠️ Runway 首页纯营销，内页（app.runwayml.com）才是工具 UI 参考
- ⚠️ Runway 的"几乎无色"对于管理型工具可能太极端，需要保留紫蓝主色做功能引导

## 4. 剪映（CapCut）详细分析

> 来源：https://www.capcut.cn/ + https://www.capcut.com/ + 使用指南

### 4.1 整体定位
- **Slogan**: "剪映AI 创作无限新可能"
- **目标用户**: 大众创作者、短视频创作者
- **核心卖点**: 模板驱动 + AI 智能功能（字幕、配音、一键成片）

### 4.2 色彩方案
- **主背景**: 深黑/深灰（桌面端编辑器）
- **主色**: 字节品牌黑 + 亮色按钮（蓝/红/橙 accent）
- **卡片**: 深灰底 + 微圆角
- **文字**: 白色标题 + 灰色次级文字
- **Accent**: 功能用多彩色（AI功能用紫色，字幕用蓝色，特效用橙色）

### 4.3 典型布局（桌面端）
- **左上**: 素材库/功能区 — 宽面板，含标签页（音频/文本/贴纸/转场/AI）
- **中上**: 播放预览区 — 16:9 比例监视器
- **右上**: 属性面板 — 参数调整、AI功能开关
- **底部**: 时间轴 — 多轨道编辑器，拖拽操作
- **大按钮**: "导出" 在右上角，醒目的亮色

### 4.4 交互特色
- 拖拽式操作为主
- AI 功能用图标+中文标签（智能字幕、智能配乐、一键成片）
- 模板中心用大图卡片（9:16竖屏模板 + 16:9横屏模板）
- 滤镜/特效用缩略图网格选择
- 快捷键 B（刀片）/ A（箭头），专业剪辑师肌肉记忆

### 4.5 中文化
- 全中文界面，专业术语也中文（蒙版、关键帧、调色、色轮）
- 功能区用图标+中文双行标签
- Solo 模式名称英文，但括号内附中文

### 4.6 对漫剧 OS 的启示
- ✅ **四区编辑器布局** — 素材/预览/属性/时间轴 — 可借鉴为分镜编辑器
- ✅ **AI 功能醒目标识** — 紫色/蓝色标记 AI 功能
- ✅ **全中文 + 专业术语** — 连"蒙版""色轮"都中文化了
- ✅ **模板卡片大图** — 视觉驱动选择
- ⚠️ 剪映是编辑器，漫剧 OS 更偏管理平台 — 布局不能照搬
- ⚠️ 剪映偏"大众创作"风格，色彩丰富活泼，不适合漫剧 OS 的工业感

## 5. 即梦（Dreamina）详细分析

> 来源：https://dreamina.capcut.com/ + 即梦智能画布功能详解

### 5.1 整体定位
- **Slogan**: "Turn inspiration to art"

- **核心卖点**: 智能画布 + AI图生图/文生图 + Seedance 视频生成

### 5.2 色彩方案
- **主背景**: 深紫黑渐变（#0D0B1E → #1A1040）
- **主色**: 紫色系渐变按钮（#7B61FF → #5B3FCC）
- **Accent**: 蓝色/青色光效（生成进度条、AI 标记）
- **卡片**: 深灰底 + 圆角 + 微妙阴影
- **文字**: 白色标题 + 浅紫灰正文

### 5.3 布局特征
- **首页**: 全幅 AI 生成图片 Hero + 渐变叠加文字
- **编辑器**: 左侧工具面板(AI生成/画笔/图层) + 中间画布 + 右侧参数面板
- **模板页**: 圆角大图卡片（角色设计/时尚美妆/游戏资产/营销广告等分类）
- **导航**: 顶部 Tab（Tools / Create / Resources）+ 语言切换 + 登录

### 5.4 交互特色
- AI 生成用紫色渐变按钮（"Try it now"）
- 模板分类用大图 + 标签 + 行动按钮
- 智能画布支持：多图层、局部重绘、抠图、扩图、轮廓边缘控制
- 风格化预览效果（卡通/3D/写实）

### 5.5 对漫剧 OS 的启示
- ✅ **紫黑渐变深色底** — 和 AiDrama 同源但更鲜艳
- ✅ **AI 功能用渐变强调** — 生成按钮/进度条明显区分
- ✅ **智能画布三栏布局** — 工具/画布/参数 — 可借鉴分镜编辑器
- ✅ **模板卡片分类展示** — 项目列表可参考
- ⚠️ Dreamina 偏"消费级创作工具"，色彩比漫剧 OS 需要的更活泼

## 6. Midjourney 分析

> 来源：产品页面 + 社区描述

### 6.1 整体定位
- **Slogan**: 无显式 slogan，走 Discord 社区模式
- **目标用户**: 艺术家、设计师、AI 爱好者
- **核心卖点**: 艺术级图像生成、风格控制

### 6.2 色彩方案
- **Discord 模式**: 深灰 #313338 背景 + 侧边栏
- **Web 模式** (alpha): 极深黑底 #000 + 白色图片
- **Accent**: 极少使用色彩，偶尔蓝绿色链接
- **图片**: 网格4张，白色/浅灰边框分隔

### 6.3 排版与交互
- Prompt 输入框是核心交互
- 图片以 2x2 网格展示（四象限图）
- U/V 按钮行内排列
- 几乎零装饰，纯内容驱动

### 6.4 对漫剧 OS 的启示
- ✅ **图片网格展示** — 版本对比可参考 2x2/4宫格
- ✅ **极简操作** — 以内容为主，UI 极度克制
- ⚠️ Discord 模式不适合生产管理工具 — 不借鉴导航模式
- ⚠️ MJ 是图像浏览器，不是管理平台 — 布局差异大

## 7. 综合建议：漫剧 OS 的 UI 风格方向

### 7.1 核心风格定位

**「影院级深色工业管理平台」** — AiDrama 的专业感 + Runway 的极简克制 + 剪映的功能密度

### 7.2 色彩方案建议

| 元素 | 当前值 | 建议调整 | 理由 |
|------|--------|---------|------|
| 主背景 | `#0A0A0A` | `#0A0A0A` 保持 ✅ | 已足够深，匹配 Runway/AiDrama |
| 卡片底 | `#141414` | `#141414` 保持 ✅ | 深灰层差，Runway 用 #1A1A1A |
| 主色 | `#7C3AED` | `#7C3AED` 保持，但**降低使用面积** | 只在 CTA 按钮、选中态、小面积标记 |
| 次级文字 | `hsl(0 0% 64%)` | 改为冷灰 `#767D88` (Runway Cool Slate) | 更冷静、更专业 |
| 边框 | `hsl(0 0% 15%)` | 改为 `#27272A` (Runway Border Dark) | 更不可见，UI 退让 |
| Hover/Active | 背景变浅色 | 轻微亮度提升 + 主色左边线 | Runway 式极简 hover |

### 7.3 排版建议

| 元素 | 当前 | 建议 | 参考 |
|------|------|------|------|
| 页面标题 | 缺统一规范 | 36px, weight 600, line-height 1.1, tracking -0.5px | Runway Section Heading |
| 区块标题 | 缺 | 20px, weight 500, line-height 1.2 | Runway Feature Title |
| 导航分组标签 | `11px uppercase` | 保持，但用 **0.35px letter-spacing** 更精细 | Runway Caption Label |
| 导航项文字 | `14px` | `15px`，微增可读性 | 剪映式实用性 |
| 项目卡片标题 | 缺 | `18px`, weight 500, line-height 1.2 | AiDrama 项目名 |

### 7.4 组件风格建议

**Sidebar（侧边栏）**
- 参考 Runway App 内页：极窄边框 + 不可见分割线 + 冷灰分组标签
- 导航项：图标 16px + 文字 15px，hover 用 `bg-white/5` 而非 `bg-muted`
- 选中态：左边线 2px `#7C3AED` + 轻底色 `bg-primary/8`
- Logo 区：发光渐变 icon（紫蓝微光）+ "漫剧 OS" 白字

**项目卡片**
- 参考 AiDrama 首页：大图封面 + overlay 标题 + 微妙渐变遮罩
- 4:3 或 16:9 宽画幅封面优先（不用正方形）
- 底部信息行：标题(白) + 状态徽标(主色) + 时间(冷灰)
- hover: 轻微 scale(1.02) + 边框变 `#27272A`

**状态徽标**
- 用小圆点(8px) + 单行文字，不用药丸型 tag
- 绿: `#22C55E` 运行中 | 黄: `#F59E0B` 等待 | 红: `#EF4444` 失败 | 蓝: `#06B6D4` 完成

**按钮**
- 主按钮: `bg-primary text-white` 紫蓝底 → 保持
- 次按钮: `border border-[#27272A] text-white/80` — Runway 式几乎看不见的边框按钮
- 文字按钮: `text-white/60 hover:text-white/90` — 极简
- 圆角: 6-8px（不是药丸形，不是 sharp）

**空状态**
- 不用大图标+长文案的 SaaS 经典空状态
- 用一行冷灰文字 (`#767D88`) + 微妙 icon，居中

### 7.5 布局建议

**Dashboard 首页**
- 顶部：大号欢迎文字 + 快速操作卡片（最近项目 / 新建项目）
- 不用 KPI 仪表盘式数字，用项目进度卡片行
- 参考 Runway 首页的留白感：区与区之间 48-64px 间距

**项目列表页**
- 大图网格（3列 or 2列宽幅），不用表格式列表
- 顶部筛选栏极简：搜索框 + 状态过滤 + 排序
- 参考 AiDrama 的项目封面展示：影视级大图 + 角标信息

**设置页**
- Tab 导航保持，但 tab 底线用 `#7C3AED` 细线（2px）而非填充
- API Key 卡片用 `#1A1A1A` 底 + 不可见边框 `#27272A`
- 输入框用深底(`#0A0A0A`) + 细边框(`#27272A`) + 聚焦时边框变主色

### 7.6 动画与过渡

- 极简：所有过渡 `transition-all duration-200 ease-out`
- 不使用渐变成型动画、弹跳、旋转等"AI 风格"效果
- 页面转场：fade，200ms
- hover 效果：scale(1.02) 最多，200ms
- 卡片进入：无 stagger 动画，直接渐入

## 8. 不做的事（红线）

1. ❌ 不做节点编辑器式画布 UI（白板/Figma 式连接线）— 管理界面状态流优于画布
2. ❌ 不做渐变紫蓝装饰条/背景 — 主色只用于功能引导（按钮/选中态），不做大面积渐变
3. ❌ 不做药丸形按钮/圆角过大的组件 — Runway 的 4-8px 圆角更专业
4. ❌ 不做弹跳/旋转/粒子动画 — 极简过渡
5. ❌ 不做亮色模式 — 纯暗色，不预留亮色切换
6. ❌ 不在侧边栏放头像/个人信息 — 工业工具不需要社交元素
